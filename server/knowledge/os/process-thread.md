# 进程与线程

## 进程的概念

进程是程序在某个数据集合上的一次执行过程，是系统进行资源分配和调度的基本单位。每个进程拥有独立的虚拟地址空间、全局变量、文件描述符表等资源。操作系统通过进程控制块（PCB）来管理进程，PCB 中保存了进程标识符 PID、进程状态、寄存器上下文、调度信息等。

进程的基本状态有三种：就绪、运行、阻塞。状态转换由调度器或事件触发：就绪→运行（被调度）、运行→就绪（时间片用完）、运行→阻塞（等待 I/O）、阻塞→就绪（I/O 完成）。

## 线程的概念

线程是进程内的一个执行单元，是 CPU 调度的最小单位。一个进程可以包含多个线程，它们共享进程的地址空间、堆区、全局变量和文件描述符，但每个线程拥有独立的栈、程序计数器和寄存器上下文。

线程分为用户级线程和内核级线程。用户级线程由用户态库管理，切换速度快但对内核不可见；内核级线程由内核直接管理，可被调度到多核并行执行。

## 进程与线程的区别

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立地址空间 | 共享进程资源 |
| 开销 | 创建/切换开销大 | 创建/切换开销小 |
| 通信 | 需 IPC 机制 | 直接读写共享变量 |
| 安全性 | 一个进程崩溃不影响其他 | 一个线程崩溃可能导致整个进程崩溃 |

## 进程间通信（IPC）

1. **管道（Pipe）**：半双工字节流，父子进程间使用。命名管道（FIFO）可用于无亲缘关系进程。
2. **消息队列**：内核中维护的消息链表，克服了管道的无格式字节流缺点。
3. **共享内存**：多个进程映射同一块物理内存，速度最快，需配合同步机制。
4. **信号量**：用于进程间同步，本质是计数器，P/V 操作原语。
5. **套接字（Socket）**：可用于跨主机通信。

## 线程同步

- **互斥锁（Mutex）**：保护临界区，同一时刻只允许一个线程进入。
- **条件变量（Condition Variable）**：线程等待某个条件成立后唤醒，常与互斥锁配合使用。
- **读写锁**：允许多读单写，提高读多写少场景的并发度。
- **自旋锁**：忙等待，适用于临界区极短的情况。

## 代码示例（POSIX 线程）

```c
#include <pthread.h>
#include <stdio.h>

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
int shared_counter = 0;

void* worker(void* arg) {
    for (int i = 0; i < 100000; i++) {
        pthread_mutex_lock(&mutex);
        shared_counter++;      // 临界区
        pthread_mutex_unlock(&mutex);
    }
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, worker, NULL);
    pthread_create(&t2, NULL, worker, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("counter = %d\n", shared_counter);  // 期望 200000
    return 0;
}
```

如果不加锁，由于 `shared_counter++` 不是原子操作（读-改-写），会出现数据竞争，结果小于 200000。

## 进程创建示例（fork）

```c
#include <unistd.h>
#include <stdio.h>

int main() {
    pid_t pid = fork();
    if (pid == 0) {
        printf("子进程 PID=%d, 父进程=%d\n", getpid(), getppid());
    } else if (pid > 0) {
        printf("父进程 PID=%d, 子进程=%d\n", getpid(), pid);
    } else {
        perror("fork 失败");
    }
    return 0;
}
```

`fork()` 采用写时复制（Copy-On-Write）技术，子进程初始时共享父进程的物理页，仅当某一方写入时才复制对应页面，从而大幅减少创建开销。

## 常见考点

1. 进程与线程的本质区别（资源分配 vs 调度单位）。
2. 进程三种状态及转换条件。
3. 写时复制（COW）原理及其对 fork 性能的意义。
4. 各种 IPC 方式的优缺点与适用场景。
5. 互斥锁、条件变量、信号量的区别和使用方式。
6. 死锁产生的四个必要条件：互斥、占有并等待、不剥夺、循环等待。
7. 银行家算法与死锁预防、避免、检测、解除策略。
8. 生产者-消费者问题、读者-写者问题、哲学家就餐问题等经典同步问题。
