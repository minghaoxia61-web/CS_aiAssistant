# 内存管理

## 虚拟内存

虚拟内存是操作系统提供的一种内存抽象技术，它使得每个进程都拥有自己独立的、连续的虚拟地址空间，而实际物理内存由操作系统统一管理。虚拟内存通过页表映射到物理内存，并可将暂时不用的页面换出到磁盘（swap），从而让程序可以使用比物理内存更大的地址空间。

虚拟内存的优势：
- 进程隔离，提升安全性；
- 简化内存分配（每个进程看到统一的地址空间）；
- 提高内存利用率，按需调入页面；
- 支持大于物理内存的程序运行。

## 分页机制

分页将虚拟地址空间划分为固定大小的页（Page，常见 4KB），物理内存划分为相同大小的页框（Frame）。MMU（内存管理单元）通过页表将虚拟页号映射到物理页框号。

虚拟地址结构：`[页号 | 页内偏移]`。例如 32 位系统、4KB 页大小：高 20 位为页号，低 12 位为偏移。

为减少页表占用空间，采用**多级页表**：将页表分级存储，仅常驻需要的部分。x86-64 采用四级页表。

为加速地址翻译，CPU 引入 **TLB（Translation Lookaside Buffer）**，即页表缓存，缓存近期使用的页表项。

## 分段机制

分段按程序的逻辑结构划分（代码段、数据段、栈段），每段大小可变。段表项记录段基址和段长，便于共享和保护，但容易产生外部碎片。

**段页式**结合二者优点：先分段，段内再分页，兼顾逻辑性和内存利用率。

## 页面置换算法

当物理内存不足需要换出页面时，使用页面置换算法决定淘汰哪一页。

### FIFO（先进先出）

淘汰最先进入内存的页面。实现简单，但存在 **Belady 异常**：增加物理页框数反而导致缺页率上升。

### LRU（最近最少使用）

淘汰最长时间未被访问的页面，性能接近最优算法 OPT。实现可用链表+哈希表，每次访问将页面移到链表头部，淘汰尾部。代价是每次访问都要更新数据结构。

### LFU（最近最少使用频率）

淘汰访问次数最少的页面。LRU 关注时间局部性，LFU 关注频率。LFU 可能因历史高频页面长期占用而淘汰新有用页面，需引入衰减机制。

### OPT（最佳置换）

淘汰未来最长时间不被访问的页面，理论最优但无法实现，仅作为对比基准。

### 示例

假设页面访问序列为 `7 0 1 2 0 3 0 4`，物理页框数为 3：

- FIFO：缺页次数 6 次
- LRU：缺页次数 5 次
- OPT：缺页次数 4 次

## 内存映射（mmap）

`mmap` 将文件或设备映射到进程虚拟地址空间，对映射区域的读写等同于对文件的操作，避免显式的 `read/write` 系统调用，提高 I/O 效率。也是进程间共享内存的底层机制。

```c
#include <sys/mman.h>
#include <fcntl.h>
#include <stdio.h>
#include <unistd.h>
#include <string.h>

int main() {
    int fd = open("test.txt", O_RDWR | O_CREAT, 0644);
    ftruncate(fd, 4096);
    char* map = mmap(NULL, 4096, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (map == MAP_FAILED) { perror("mmap"); return 1; }

    strcpy(map, "Hello, mmap!");      // 写入映射区域即写入文件
    printf("读取: %s\n", map);

    munmap(map, 4096);
    close(fd);
    return 0;
}
```

## 内存分配示例（malloc 简化）

用户态的 `malloc` 通常通过 `brk`/`mmap` 向内核申请内存，再由用户态分配器（如 glibc 的 ptmalloc）切分给应用。小于 128KB 的请求一般用 `brk` 扩展堆顶，大于则用 `mmap` 申请独立映射区。

## 常见考点

1. 虚拟内存的概念与作用。
2. 分页地址翻译过程（页号、偏移、页表、TLB）。
3. 多级页表为什么能节省空间。
4. TLB 的工作原理与命中率影响。
5. FIFO 的 Belady 异常，LRU 与 OPT 的对比。
6. 缺页中断的处理流程。
7. 内部碎片与外部碎片的区别。
8. `mmap` 与 `read/write` 的对比，零拷贝技术。
9. 伙伴系统、slab 分配器的工作机制。
