# OSI 与 TCP/IP 模型

## 网络分层思想

网络协议采用分层架构，每层完成特定功能，下层为上层提供服务，层间通过接口（SAP）交互。分层的好处：各层独立、灵活性好、易于实现和维护、促进标准化。

## OSI 七层模型

OSI（Open Systems Interconnection）参考模型由 ISO 提出，从下到上共七层：

| 层次 | 名称 | 主要功能 | 典型协议 |
|------|------|----------|----------|
| 7 | 应用层 | 为用户应用程序提供网络服务 | HTTP, FTP, SMTP, DNS |
| 6 | 表示层 | 数据格式转换、加密压缩 | JPEG, SSL/TLS |
| 5 | 会话层 | 建立、管理、终止会话 | RPC, NetBIOS |
| 4 | 传输层 | 端到端可靠/不可靠传输 | TCP, UDP |
| 3 | 网络层 | 路由选择与逻辑寻址 | IP, ICMP, OSPF |
| 2 | 数据链路层 | 相邻节点间的帧传输 | Ethernet, PPP, ARP |
| 1 | 物理层 | 比特流的透明传输 | RJ45, 光纤, Wi-Fi |

记忆口诀（自下而上）："物数网传会表应"。

各层核心职责：
- **物理层**：定义电气、机械、过程规范，传输比特流。
- **数据链路层**：将比特组装成帧，处理差错检测（CRC）、MAC 地址寻址、流量控制。
- **网络层**：负责分组转发和路由，IP 地址寻址，实现异构网络互联。
- **传输层**：提供进程间通信，TCP 保证可靠有序，UDP 提供轻量传输。
- **会话层**：管理不同应用之间的会话同步。
- **表示层**：统一数据表示，处理加解密、压缩。
- **应用层**：直接面向用户，定义应用协议语义。

## TCP/IP 四层模型

实际互联网采用的体系结构，分为四层：

| TCP/IP 层 | 对应 OSI 层 | 典型协议 |
|-----------|-------------|----------|
| 应用层 | 应用 + 表示 + 会话 | HTTP, DNS, SMTP |
| 传输层 | 传输层 | TCP, UDP |
| 网际层 | 网络层 | IP, ICMP, ARP |
| 网络接口层 | 数据链路 + 物理层 | Ethernet, Wi-Fi |

TCP/IP 模型将 OSI 的上三层合并为应用层，下两层合并为网络接口层，更贴近实际实现。也有教材把它看作五层模型（把网络接口层拆开），方便教学。

## 两种模型的对比

| 对比项 | OSI | TCP/IP |
|--------|-----|--------|
| 层数 | 7 层 | 4 层 |
| 网络层 | 无连接 | 无连接（IP） |
| 传输层 | 面向连接 + 无连接 | TCP 面向连接 / UDP 无连接 |
| 出现时间 | 先有理论 | 先有协议后有模型 |
| 实际应用 | 仅作教学参考 | 互联网实际标准 |

## 数据封装过程

发送方自上而下逐层封装，每层添加自己的首部（链路层还加尾部）；接收方自下而上逐层解封装。

```
应用层数据 (Data)
   ↓ 加 TCP/UDP 头
段 (Segment) / 数据报 (Datagram)
   ↓ 加 IP 头
分组 (Packet)
   ↓ 加帧头帧尾
帧 (Frame)
   ↓ 转换为比特流
比特 (Bits) 经物理介质传输
```

各层数据单元名称：
- 应用层：报文 Message
- 传输层：段 Segment（TCP）/ 数据报 Datagram（UDP）
- 网络层：分组 Packet
- 链路层：帧 Frame
- 物理层：比特 Bit

## PDU 与 SDU

每层接收上层的 SDU（服务数据单元），加上本层控制信息 PCI 后形成 PDU（协议数据单元），再交给下层作为新的 SDU。这是分层服务模型的核心抽象。

## 抓包示例

使用 Wireshark 抓取一个 HTTP 请求，可以看到以太网帧头 → IP 头 → TCP 头 → HTTP 数据层层嵌套，正是封装过程的直观体现。

## 常见考点

1. OSI 七层名称、顺序及各层功能（背诵 + 理解）。
2. TCP/IP 四层与 OSI 七层的对应关系。
3. 数据封装过程中各层 PDU 的名称。
4. 哪些设备工作在哪一层：集线器（物理层）、交换机（链路层）、路由器（网络层）、网关（应用层）。
5. ARP 协议工作层次争议（介于链路层与网络层之间）。
6. TCP/IP 与 OSI 哪个先出现、为什么 OSI 没能成为主流。
7. 五层模型为何适合教学。
8. 同层通信的本质：逻辑上的对等通信，实际数据流垂直穿过协议栈。
